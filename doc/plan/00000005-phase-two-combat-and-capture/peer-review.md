# Peer Review — Story 00000005: Phase 2 combat (attacks & capture resolution)

## Summary

This branch adds Phase 2 combat by extending — not forking — story 00000004's
model: a new pure `combat.ts` (`resolveCombat` → `CombatOutcome`) implements the
full ruleset-1.1 rank table and every special case (Knight charge, Assassin both
directions incl. the Tower exception, Sapper-vs-Tower, Halberdier anti-charge,
Knight-vs-Knight, and Archer defensive support); `movement.ts` gains
`legalAttacks`; `play.ts`'s `applyMove` now resolves attacks and returns a
`{ state, outcome }` pair exposing a discriminated `PlyOutcome` for story
00000006; and the session/announcement/board layers offer, distinguish, apply,
and announce attacks through the existing accessible-grid interaction. The
combat rule logic is correct against the story's ruleset facts, immutability and
the throw-on-illegal-move invariant are preserved, and attacks record as plain
`A2A3`. `npm run typecheck`, `npm run lint`, and `npm test` (214 tests, 14
files) all pass; the two findings below are minor.
