# Peer Review — Story 00000014: Import and review a recorded game

## Summary

This branch puts a start screen in front of the app and adds a complete
import-and-review path: a versioned record reader (`notation.ts`,
`parsePositionBlock`, `recordFile.ts`, `replay.ts`, dispatched by
`src/rules/readRecord.ts`), player-facing wording (`src/review/reviewText.ts`),
a cursor-based review session, and the review screen (inert shared board, step
and jump controls, clickable move list, recorded result), plus a
confirm-before-leaving dialog for a hot-seat game in progress. The
presentational board was extracted from `PlayBoard.tsx` into `FullBoard.tsx` so
the reviewer shares it without depending on `PlaySession`, and the hot-seat game
moved verbatim into `HotSeatGame.tsx`. The plan's 16 steps are all marked
committed and each carries Notes; the plan does include a README-verification
step (Step 16), and `README.md` was updated accordingly.

Verification run from the repository root: `npm run typecheck` **passes**;
`npm run lint` **passes** (no findings); `npm test` **passes** (26 files, 477
tests). The findings below come from reading the diff against the story and the
plan.
