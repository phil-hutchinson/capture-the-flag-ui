# Peer Review — Play against the engine (baseline)

## Summary

This branch adds a third start-screen mode — play against the computer — with a
full vertical slice: an ENG_NN_1 board encoder (`src/encoding/eng-nn-1/`),
in-browser ONNX inference via onnxruntime-web (`src/engine/inference.ts`), a
legal-masked policy decoder + sampler, an async engine-player seam, and a new
`EngineGame` screen that reuses the existing placement/play components. The
implementation tracks the story's fixed policy and the plan's steps closely,
with the single planned deviation (two additive `PlayBoard` props) documented in
the Step 5 Notes; the engine-drives-the-rules invariant is preserved
structurally (only engine-legal plies are ever scored).

**Re-review (Step 9 — animate the computer's move).** A follow-up step, added to
the plan post-sign-off on owner feedback, was committed since the last review
(`c585070`). It slides the computer's just-applied piece from origin to
destination over `MOVE_SLIDE_DURATION_MS` (333 ms) via a pure-CSS `@keyframes`
overlay in a new `.full-board__stage` wrapper, highlights the move's path
(`from`/between/`to`) by reusing the `--destination` amber fill through a new
pure `movePathSquares` helper, keeps the board inert for the slide's duration,
and honors `prefers-reduced-motion`. The Step 5/6 stale-move and StrictMode
guards are intact: the `cancelled` flag is still checked before the new
`setAnimatedMove` call, and the slide-clear timer lives in a separate
`animatedMove`-keyed effect — a documented, correctly-reasoned deviation from
the plan's "same effect" wording (folding it into the computer-turn effect would
have cleared the timer on the very render that starts the slide). The
`.full-board__stage` wrapper is a flex item under `.app__board-column` in all
consumers, so its `inline-block` is blockified and hot-seat/review are visually
unaffected. The geometry (`fullBoardDisplayPosition`) and path helper are pure,
correct for both orientations, and unit-tested. No correctness defects were
found in Step 9; two Minor observations are appended below (#5, #6). Prior
findings #1–#3 (in `inference.ts`) and #4 (in `EngineGame.tsx`'s placement
handlers) are not touched by Step 9 and remain Resolved / Won't fix exactly as
recorded.

Verification (run from the repo root against the current branch):

- `npm run typecheck` — **pass** (no errors).
- `npm run lint` — **pass** (no findings).
- `npm test` — **pass** (28 files, 491 tests passing).

The findings below are all Minor: maintainability and small-cleanliness

found; the story's in-scope items and manual gates are all wired, and the plan
includes an explicit README-verification step (Step 8, re-confirmed unchanged by
Step 9), which was carried out.
