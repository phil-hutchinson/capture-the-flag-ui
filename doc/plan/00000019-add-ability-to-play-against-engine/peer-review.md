# Peer Review — Play against the engine (baseline)

## Summary

This branch adds a third start-screen mode — play against the computer — with a
full vertical slice: an ENG_NN_1 board encoder (`src/encoding/eng-nn-1/`),
in-browser ONNX inference via onnxruntime-web (`src/engine/inference.ts`), a
legal-masked policy decoder + sampler, an async engine-player seam, and a new
`EngineGame` screen that reuses the existing placement/play components. The
implementation tracks the story's fixed policy and the plan's steps closely,
with the single planned deviation (two additive `PlayBoard` props) documented in
the Step 5 Notes; the engine-drives-the-rules invariant is preserved
structurally (only engine-legal plies are ever scored).

Verification (run from the repo root against the current branch):

- `npm run typecheck` — **pass** (no errors).
- `npm run lint` — **pass** (no findings).
- `npm test` — **pass** (28 files, 481 tests passing).

The findings below are all Minor: maintainability and small-cleanliness
observations. No correctness, contract, or requirements-coverage defects were
found; the story's in-scope items and manual gates are all wired, and the plan
includes an explicit README-verification step (Step 8), which was carried out.
