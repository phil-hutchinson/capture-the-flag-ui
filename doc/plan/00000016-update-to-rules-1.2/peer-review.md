# Peer Review — Story 00000016: Update to rules 1.2

## Summary

This branch replaces the app's ruleset implementation with companion-repo
version 1.2: the folder rename `v1_1` → `v1`, the 25-piece roster with renames
and the Halberdier/Foot Soldier rank swap, sparse placement with the
Tower-adjacency rule, the one-square/unencumbered-two-square movement model,
the rank/formation/Tower/Flag combat model, the single shared inactivity-draw
ending, and the `1.2:PRE-RELEASE` record tag. `npm run typecheck`, `npm run
lint`, and `npm run test` (428 tests, 24 files) all pass. The implementation
faithfully tracks the plan and rules text in nearly every respect; the one
substantive finding is that the inactivity counter does not reset on a
complete-sacrifice attack, contrary to the source-of-truth rules and story.md.
